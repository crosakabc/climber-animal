## 1. Installing MelonLoader

> **Note**: MelonLoader is already included in the current folder — normally you do not need to reinstall.  
> Simply browse to the local game directory and place the MelonLoader program folder into the game root directory.

### Compatibility Notes

The current Climber Animals Together - AsianMode version (latest: 2026/3/30) can still use this MOD tool after updates.

**Troubleshooting flow when errors occur**:

1. Try configuring the `.cs` file and recompiling (**be sure to back up the original release files**)
2. Restart the game
3. If the issue persists, delete MelonLoader and reinstall the latest version
4. Restart the game — this usually resolves the problem

### Fresh Installation Steps

#### 1.1 Download MelonLoader Official Installer

- Download link: https://github.com/LavaGang/MelonLoader/releases/download/v0.7.2/MelonLoader.Installer.exe

#### 1.2 Run the Installer

Right-click `MelonLoader.Installer.exe` → **Run as administrator**

#### 1.3 Select the Game Executable

Click **Select** and choose the game executable — it will auto-detect `Climber Animals Together`

#### 1.4 Click Install

Click **Install** and the installer will automatically:

- Download the correct version of Cpp2IL (including all dependency DLLs and a complete Plugins folder)
- Auto-generate `Mods`, `Plugins`, `UserData` and other folders

### Reinstallation Notes

The install plugin generates several files on startup. Subsequent reinstalls may throw errors — handle as follows:

1. Use AI to troubleshoot the error
2. If unresolved, delete all plugin-generated files
3. Re-run the exe installer to regenerate required dependency files

#### Classic Error Handling

> **If you see errors after closing the game and plugin → delete all plugin-generated files → re-run the exe installer**

- If you see **two indicators**, some files were missed — cannot reinstall yet
- If you see **one indicator**, you can proceed with reinstallation

**Generally, the MOD will be recognized normally after a clean reinstall and restart.**

### Original Game Files Reference

Below are the original game files (**do not delete these**). All other files are MelonLoader-generated and can be safely deleted:

> Keep the original files in the game install directory. MelonLoader-generated folders (Mods, Plugins, UserData, etc.) can be deleted and regenerated by reinstalling.

## 2. About Mods Files

### 2.1 File Placement

1. Ensure `AnimalTransferMod.dll` is placed in the **Mods** folder under the game root directory
2. Make sure `AnimalTransferMod_Save.txt` exists in the Mods folder for saving coordinate data (included in the provided folder)

**Example path**:

```
D:\steam\steamapps\common\Climber Animals Together\Mods\
```

### 2.2 Managing Other Mod Files

- Other Mod files can remain in the `Mods` folder
- Alternatively, place them alongside `AnimalTransferMod.dll` in a self-created `release` folder as a backup

## 3. Implemented Features

### Teleport GUI Menu

| Shortcut | Function |
|----------|----------|
| **F4** | Open / Close teleport GUI menu |

### Feature List

| # | Feature | Description |
|---|---------|-------------|
| 1 | **Real-time player coordinates** | View all online players' positions in real time |
| 2 | **Teleport self to target** | Teleport your character to a target player's position |
| 3 | **Teleport target to self** | Teleport a target player to your position |
| 4 | **Custom coordinate teleport** | Enter custom X/Y/Z coordinates for precise teleportation |
| 5 | **Position save / rollback** | `Ctrl+S` to save / `Ctrl+Z` to rollback |

### ⭐ Core Highlight: Position Save / Rollback

> A **reliable position rollback feature** has been implemented, allowing for theoretically unlimited retries **without affecting achievement unlocks**.

- **Save**: Press `Ctrl+S` to save your current character position to `AnimalTransferMod_Save.txt`
- **Rollback**: Press `Ctrl+Z` to read the last saved coordinates and teleport your character back to that position

## 4. Known Bugs

| # | Issue | Workaround |
|---|-------|------------|
| 1 | May **immediately flash back** after teleport due to network issues | Try the teleport function multiple times |
| 2 | Character may occasionally **get stuck** after rollback teleport | Try jumping — this usually recovers |
| 3 | Keeping the GUI window open causes **stuttering** | Do not keep the GUI open (coordinate polling runs per-frame) |
